# 实时检测系统 - 解码-处理-编码流水线架构说明

本系统采用**解码-处理-编码流水线架构**与**事件驱动微服务**，各模块通过**队列**解耦，实现检测结果与视频画面**完美同步**和**低延迟**观看体验。

---

## 流水线架构概览

整体为 **「解码 → 处理 → 编码 → 传输」** 四阶段：

1. **解码**：视频流接入模块接收原始视频流（如 RTSP），解码为图像帧并写入 `frame_queue`。
2. **处理**：检测引擎对每一帧进行 AI 分析，展示服务将检测结果（物体框、标签）**直接绘制在该帧图像上**，输出已标注帧。
3. **编码**：编码服务将已标注帧重新编码为新的视频流（HLS），写入 `hls_output_dir`。
4. **传输**：带检测框的新视频流通过**低延迟协议（HLS）** 由 Web 服务器提供，前端优先使用 HLS 播放，失败时回退 MJPEG，确保检测结果与画面同步、观看流畅。

## 模块与包对应关系

| 架构角色           | 实现位置                     | 说明 |
|--------------------|------------------------------|------|
| 事件类型定义       | `core/events.py`             | FrameEvent、DetectionResultEvent、AlertEvent |
| 解码 - 视频流接入  | `ingestion/stream_service.py` | 接收 RTSP/本地视频，解码为帧，写入 frame_queue |
| 处理 - 检测引擎    | `detection/*_detector.py`    | 插件化模型池，BM1684X 推理，结果写入 result_queues |
| 处理 - 展示服务    | `core/display_service.py`   | 汇总结果并**在帧上绘制检测框**，写入 output_queue + encoder_queue |
| 编码 - 编码服务    | `core/encoder_service.py`   | 已标注帧 → HLS 流（live.m3u8 + segment_*.ts） |
| BM1684X 适配器     | `core/bm1684x_yolo_adapter.py` | YOLO bmodel 推理适配 |
| 存储层             | `storage/` + VideoBufferManager | 告警目录、list_alarm_files 历史回溯 |
| 传输 - Web 层      | `backend/app.py`              | Flask、HLS 静态服务、MJPEG 回退、SSE 告警 |

## 事件与队列

- **frame_queue**：解码层 → 检测引擎 / 展示服务；承载原始/解码后帧数据（含 stream_id）。
- **result_queues[detector]**：检测引擎 → 展示服务；承载各检测器的分析结果。
- **output_queue**：展示服务 → Web 层；承载已标注帧，供 MJPEG /video_feed 回退。
- **encoder_queue**：展示服务 → 编码服务；承载已标注帧，供 HLS 编码（解码-处理-编码流水线）。
- **alert_queue**：展示服务（告警保存回调）→ Web 层；承载 AlertEvent，用于 SSE 实时推送。
- **control_queues**：主控 → 各服务；启停、enable/disable 等控制命令。

各模块仅依赖队列/事件结构，不直接依赖其它进程内部实现，确保**高实时性、可扩展性和稳定性**。

## 实现功能（保持不变）

- 多路视频流（1/2/4/9/16 路）接入与宫格展示  
- 六类检测：跌倒、呼吸机、打架、聚集、安全帽、窗户门  
- 时间段自动开关检测器（TimeslotScheduler）  
- 告警前后视频/图片保存至 alarm_videos，通过 Web 端下载  
- Web 控制面板：启停、检测器开关、时间段配置、状态刷新  
- **解码-处理-编码流水线**：解码 → 检测并绘制在帧上 → HLS 编码 → 前端 HLS 低延迟播放，MJPEG 回退  
- SSE 实时告警推送、历史告警列表 API 与前端展示
